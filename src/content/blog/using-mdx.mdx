このテーマには[@astrojs/mdx](https://docs.astro.build/ja/guides/integrations-guide/mdx/)があらかじめ組み込まれており、`astro.config.mjs`にも設定が済んでいます。MDXを使わない場合は設定から該当のインテグレーションを外すだけで無効化できます。

## なぜMDXなのか

MDXはMarkdownにJavaScriptやJSXの構文を追加した拡張フォーマットです。[Markdown内でUIコンポーネントを呼び出せる](https://docs.astro.build/ja/guides/markdown-content/#mdx%E3%81%A7%E5%87%BA%E6%9D%A5%E3%82%8B%E3%81%93%E3%81%A8)ため、インタラクティブなチャートやアラート、CTAボタンなどを簡単に埋め込めます。既存のMDX資産がある場合も、Astroへの移行がスムーズになります。

## 使い方の例

MDX内でAstroコンポーネントをインポートし、そのままJSXライクな構文で描画できます。以下の例では、`HeaderLink`コンポーネントを読み込み、クリックするとアラートが表示されるリンクを挿入しています。

import HeaderLink from '../../components/HeaderLink.astro';

<HeaderLink href="#" onclick="alert('clicked!')">
        MDXで埋め込んだコンポーネント
</HeaderLink>

ブラウザでこのページを開くと、上記のリンクをクリックした瞬間にブラウザのアラートが表示されます。AstroはMDX内のコンポーネントも事前レンダリングしますが、動的な振る舞いをさせたい場合は次の項目を確認してください。
